# 現状の業務フロー（清掃管理）

## "磁石とシール"で動くアナログ現場

```
【現在の清掃フロー】

チェックアウト
  ↓
鍵がフロントに返却
  ↓
部屋番号の磁石シートを清掃リーダーに手渡し（物理的に）
  ↓
清掃リーダーが各フロアの扉に磁石を貼付
  ↓
清掃完了後、担当者名で記録（手書き）
```

**問題点：**
- どこが済んでいるか全体把握が困難
- 抜け漏れが起きても気づきにくい
- コンピューターが介在していない

# 課題①：清掃計画作成の負担

## 山本さんの毎日の「パズル作業」

**現状の作業内容（毎朝）：**

1. ホテルシステムの画面を印刷または目視確認
2. チェックアウト・連泊・エコクリーニングを**一部屋ずつ手動で分類**
3. 各清掃スタッフに部屋を割り当て（複雑なルールあり）

**割り当てルール（頭の中でパズル）：**
| 部屋タイプ | 清掃係数 |
|------------|---------|
| シングル   | 1.0     |
| ダブル     | 1.1〜1.2 |
| ツイン     | 1.5     |
| トリプル   | 2.0     |

- 1スタッフの上限：**1日10部屋**
- 同一フロア優先（移動コスト最小化）
- 連泊4泊目に清掃対応など特殊ルールあり

# 課題②：シフト管理・スタッフ調整

## 前日の手動アサイン＋月次シフト作成

**毎日の調整：**
- 前日夜に翌日の予約状況を確認
- 宿泊が少なければフロントが清掃スタッフに個別電話してキャンセル
- 当日の急な変更にも人力対応

**月次シフト作成：**
- 15日までにスタッフから希望休暇を収集
- 20日前までに翌月シフト確定
- 休暇希望が集中した日の調整が特に大変

> **現状：** スプレッドシートで手動作成、条件ミスが翌朝発覚することも

---

# 課題③：清掃スタッフ給与計算

## 出来高制＋基本給の複雑な計算

**給与計算の仕組み：**
- 基本給：4.5時間ベース
- 出来高：清掃部屋数に応じて加算
  - 例：11部屋 → 基本給、12部屋 → 追加手当

**問題のある計算プロセス：**
1. 清掃スタッフが自己申告（「今日13部屋やりました」）
2. 社長が手動で集計・計算
3. 月間の労働日数・上限・繰越ルールなど多数の変数
4. エラーリスクが高く、確認作業に時間がかかる

---

# 課題④：リネン・備品の在庫管理

## 目視カウントで発注量を決定

**現在の流れ：**
- 清掃終了後にキーパーが目視で在庫確認
- 経験則で翌日のクリーニング発注量を算出
- 手書きメモ → 業者に電話注文

**リスク：**
- カウントミスによる在庫不足
- 過剰発注によるコスト増
- 担当者変更時の引き継ぎ困難

# 取り組みの基本方針

## 一朝一夕でいかない。だから段階的に進める

**3つのアプローチを組み合わせる：**

### ① 日々のオペレーション改善
小さな改善を積み重ね、現場に定着させる

### ② AI活用で判断・計算を自動化
繰り返しのパズル作業をAIに任せる

### ③ ツール化で仕組みを簡易化
個人の知識・経験をシステムに落とし込む

> **重要原則：** 現場が使いやすいものを作る。現場が作ったものが一番使いやすい。

# フェーズ1：清掃管理のデジタル化

## まず「見える化」から始める（着手済み）

**ターゲット：清掃WebApp（スマホ対応）**

**主な機能：**
- 部屋ステータス一覧（清掃待ち / 清掃中 / 完了 / 検査済み）
- フロア別・担当者別の進捗表示
- 自分で空き部屋に手を挙げてアサイン
- 清掃完了後の実績入力（部屋数カウント自動集計）
- フロントと清掃スタッフ双方からリアルタイム確認

**利用端末：**
- 清掃スタッフ：共用スマートフォン or 個人スマホ
- フロント：既存PC / タブレット

---

# フェーズ2：清掃計画の自動化

## ホテルシステムデータを取り込んで計画自動生成

**データ取得方法（個人情報保護対応）：**

1. ホテルシステム画面を**ワンボタンでスクリーンショット**
2. ローカルAIが個人名を**自動マスキング**
3. 部屋番号・状態情報だけをAIに渡す
4. AIが**清掃割り当て案を自動生成**

**割り当てルールの設定例：**
- 部屋タイプ別の係数（1〜2倍）
- スタッフ上限（10部屋/日）
- フロア近接優先
- 連泊ルール（4泊目清掃など）

> **個人情報対策：** クラウドAIではなく**ローカルLLM**を活用

---

# フェーズ3：給与計算・シフト管理の自動化

## 実績データからシームレスに計算へ

**清掃実績の自動集計：**
- WebAppに入力された部屋数を自動集計
- 基本給＋出来高をルールに従い自動計算
- 月末一括処理で給与明細データ出力

**シフト管理AI補助：**
- 予約状況から必要人員を予測
- スタッフの希望休暇を考慮した最適シフト案を生成
- 条件違反（上限超過など）を自動チェック

---

# フェーズ4：情報の蓄積と活用

## 現場の知恵をナレッジベースに

**蓄積すべきデータ：**
- 清掃実績（誰が・どの部屋を・何分で）
- 設備不具合報告（写真付き）
- リネン使用量と発注実績
- お客様フィードバック（じゃらん・楽天など）

**AIによる活用：**
- 繁忙期・閑散期のパターン分析
- クレーム傾向の自動抽出
- オペレーション改善提案

> **目指す姿：** 「また同じ問題が起きた」を「仕組みのせい」として解決できる組織へ
