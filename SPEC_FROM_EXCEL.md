## 2. 清掃ステータス（実際の運用）

Excelで使われていた状態値（B列・E列・H列…各フロアの「状態」列）：

| ステータス | 内容 | 対応する清掃作業 |
|-----------|------|-----------------|
| **C/O** | チェックアウト後の全清掃 | リネン全交換・全アメニティ補充 |
| **エコ** | 連泊客の簡易清掃 | タオル類補充のみ、リネン交換なし |
| **ステイ** | 清掃不要（長期連泊等） | 清掃対象外 |
| **空き部屋** | 空室（昨夜も空き） | 軽確認程度 |
| **ｱﾒﾆﾃｨ** | アメニティ補充のみ | アメニティ配置のみ |
| **繰越部屋** | 前日から持ち越した清掃対象 | C/OまたはエコのC/Oレベル |
| **繰越清掃** | 前日から持ち越した清掃作業 | 通常清掃と同等 |
| **R/C** | ルームチェンジ（部屋移動） | 入居前清掃確認 |
| **連泊** | 連泊確認済み（ステイ相当） | 清掃対象外 |
| **特殊清掃** | 脱臭・特殊対応が必要な部屋 | 通常清掃＋特殊処理 |

> **現行SPEC.mdとの差分**: 「空き部屋」「ｱﾒﾆﾃｨ」「繰越部屋」「繰越清掃」「R/C」「連泊」「特殊清掃」が未定義だった

## 4. 担当割り当てルール（スプレッドシートに明記）

### 4-1. 1人あたり清掃数の目安

```
* 1名あたり10R以下にしない（基本は11）
* キーパー最大2〜3R（下キーパーは2R以下）
* ステイ数・アウト数が同じくらいになるように均等配分
* W・Tも同じくらいで（偏りをなくす）
```

### 4-2. フロア割り当てルール

```
* フロアは「2フロア」担当で「上下階」でペアにする
  - 例: 3F と 4F を同一スタッフ / 5F と 6F を同一スタッフ
* キーパーは上下で分ける
  - 下キーパー: 2F・3F・4F
  - 上キーパー: 5F・6F・7F
```

### 4-3. 2Fの特殊扱い

```
* 8. 2Fに割り当てる清掃を順番に！
* 2F清掃順シートで清掃順序を管理（別シート）
```

### 4-4. 2名セット制度

一部の部屋（主にWタイプ）は2名を1チームとして割り当てる運用がある:
- **2名セット（継続）**: 引き続き2名で対応
- **2名セット（解除）**: 次回から1名に戻す
- 例: `SW/2名セット（解除）…414`

## 7. 13時清掃点検ルール（2025年3月27日改定）

```
13時清掃点検…
  13時・14時・15時分をすべて「13時清掃点検終了」として
  清掃管理表に記載する

運用の再確認:
  アサインは空き部屋を優先し、
  13〜15時C/Iは「13時清掃点検終了」として清掃管理表に記載する
```

> **意味**: 13〜15時チェックインの部屋は一括して「13時清掃完了」扱いにする

## 8. 日々の運用フロー（8ステップ）

Excelシート上に以下の手順が明記されていた：

```
1. 状態選択 — 各部屋の今日の状態を選択（C/O / エコ / ステイ etc.）
2. 出勤選択 — 各スタッフの出勤/休みを選択
3. 清掃数入力 — 1名あたりの清掃室数を入力
4. スタッフ選択 — 各部屋に担当スタッフを割り当て
5. 全部埋まったら印刷（1ページのみ・カラー・2部）
6. 作業指示書に転記（担当ごと・太いサインペン）
7. 作業指示票とタブレット（輪ゴム止め）をセットにする
8. 2Fに割り当てる清掃を順番に！
```

> **背景**: 現在はタブレットと紙の作業指示票を一緒にスタッフに渡している運用

## 9. 繰越（翌日持ち越し）の管理

- 当日清掃できなかった部屋を翌日に繰り越す概念がある
- 列には「繰越部屋」「繰越清掃」「翌日繰越」が別セルとして存在
- 繰越分は翌日の清掃数に加算して計算
